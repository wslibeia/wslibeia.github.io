配置Jenkins服务器自动监听Gerrit过程中碰到的问题

gonglja

2019-12-07

系统集成

在配置过程中碰到的一些问题，记录一下。

本文所有的问题出现的前提均在jenkins2.203和装有gerrit trigger环境下出现。

1 Gerrit trigger中Gerrit Servers中ssh keyfile问题

问题描述

Gerrit trigger中Gerrit Servers中ssh keyfile问题，表现形式为

1> …does not exist

2> exception.SSHException: not a valid RSA private key file

解决方案

针对1> 原因

​ 1.找不到id_rsa文件，

​ 2.没有用户权限

​ 3.没有访问权限

最终解决方案：使用jenkins服务器下的jenkins账户创建公私钥^[1]^，在创建过程中指定路径^[2]^，给.ssh文件赋权限^[3]^

针对2> 原因

​ 1.密钥格式不对，使用-m 参数创建旧格式的密钥^[1]^

ps：Gerrit Trigger中的ssh port端口非gerrit机器的ssh端口22，而是gerrit软件创建的ssh端口，具体端口号可通过命令^[4]^在gerrit查得，默认为29418。

确认以上后，基本test connection就可以success。

2 Jenkins出现No valid crumb was included in the request

问题描述

Jenkins出现No valid crumb was included in the request

==解决方案==

在jenkins 的Configure Global Security下 , 取消“防止跨站点请求伪造（Prevent Cross Site Request Forgery exploits）”的勾选。

ps：实际过程中我没有取消这个但是实际还是可以用了。

3Jenkins 没有权限监听 Gerrit 的 ‘Stream Events’的问题

问题描述

当点击 test connection时，显示

User jenkins has no capability to connect to Gerrit event stream

==解决方案==

  1. 管理员登录gerrit

  2. Projects->List->All-Projects

  3. Projects->Access

    • Global Capabilities->Stream Events 点击 Non-Interactive Users

    • 添加 账户（Jenkins@zjc.com）用户到 ‘Non-Interactive Users’ 组

4 如何配置Gerrit Trigger，使得能够监控gerrit的提交

1.新建任务

输入任务名称后，选择构建一个自由风格的软件项目，点击确定

2.打开新建的任务，点击左边的配置，选择构建触发器为Gerrit event，接下来就是配置Gerrit Trigger，

点击选择Choose a Server ,选择你上一步配置的Gerrit服务器

点击新增，选择Patchset Created并在Exclude No Code Change上面打钩

点击新增，选择Gerrit Project，如果要监听所有项目的所有分支

Project Type选择为Path，pattern为 **

Branches Type 选择为Path，pattern 为 **

若要监听单个的项目的单个分支

Project Type选择为Plain，pattern为 项目名

Branches Type 选择为Plain，pattern 为 分支名

5 如何将jenkins监控到Gerrit的提交clone到编译服务器

==关键点 使用jenkins Gerrit trigger 插件监听Gerrit，待检测到提交后通过参数传递给远程服务器==

如何配置

找到构建，点击增加构建步骤，选择Execute shell script on remote host using ssh（使用ssh在远程主机上执行shell命令），添加ssh site 选择为编译服务器，在Command中添加如下命令

​ 1>首先进入工作文件夹

​ 2>clone 下整个工程

​ 3>进入工程

​ 4>拉取指定改变

ps：注意一定不要选择 Execute each line，否则会开启多个shell窗口，导致cmd并不是按照顺序执行。

构建参数如何获取

当有一次gerrit提交触发jenkins任务后，点击该任务，找到左边Build History，点进最近的一次构建，点击左边的参数即可看到各项参数。
