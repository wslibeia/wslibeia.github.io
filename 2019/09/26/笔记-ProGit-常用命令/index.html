笔记_ProGit 常用命令

gonglja

2019-09-26

git

如何学习Git，那就从本文章开始吧。

git常用命令

本地gitlab搭建

配置用户信息

用户信息

第一个要配置的是你个人的用户名称和电子邮件地址。这两条配置很重要，每次 Git 提交时都会引用这两条信息，说明是谁提交了更新，所以会随更新内容一起被永久纳入历史记录：

如果用了 --global 选项，那么更改的配置文件就是位于你用户主目录下的那个，以后你所有的项目都会默认使用这里配置的用户信息。如果要在某个特定的项目中使用其他名字或者电邮，只要去掉 --global选项重新配置即可，新的设定保存在当前项目的 .git/config 文件里。

git config

–list 查看配置信息

生成密钥

生成密钥后 cat id_rsa.pub复制git服务器端公钥处。

本地管理

1.git init

在工作目录中初始化新仓库

2.git add <file>

将<file>添加到暂存区

3.git status

检查当前文件的状态

4.git diff

查看工作目录中当前文件和暂存区域快照之间的差异，也就是修改之后还没有暂存起来的变化内容。

参数:

–cached（it） 查看已经暂存起来的文件和上次提交时的快照之间的差异。

5.git commit

提交暂存区域的内容

参数：

-m “说明” 提交带有”说明”的内容

-a 自动把所有已经跟踪过的文件暂存起来一并提交，从而跳过git add步骤

–amend 修改最后一次提交（撤销上一次提交，然后将漏掉的文件add到暂存区域，然后在git commit）

6.git rm

从暂存区域移除，并连带从工作目录中删除指定的文件。

参数：

-f <file>删除之前修改过已经放到暂存区域的数据（防止误删除文件后丢失修改的内容）

–cached <file> 只在暂存区域移除，不删除本地工作目录

7.git mv <file_from> <file_to>

将文件<file_from>改名为<file_to>

8.git log

查看提交历史

参数：

-p 选项展开每次提交的内容差异

-2 则显示最近的两次更新

–stat 仅显示简要的增改函数统计

远程仓库的使用

9.git remote

查看当前的远程库

参数：

-v 显示对于对应的克隆地址

add [shortname] [url] 添加远程仓库（可指定一个名字，以便将来引用），用shortname指代对应的仓库地址

show [remote-name] 查看某个远程仓库的详细信息

rename [old-name] [new-name] 修改某个远程仓库在本地的简称（注 对远程仓库的重命名，也会使对应的分支名称发生变化）

10.git fetch [remote-name]

从远程仓库抓取数据到本地（到远程仓库中拉取所有本地仓库中还没有的数据）

只抓取数据，不合并分支

11.git pull

前提：设置了某个分支用于跟踪某个远端仓库的分支。

自动抓取数据到本地仓库并合并到本地仓库中的当前分支。

12.git push [remote-name] [branch-name]

将本地仓库中的数据推送到远程仓库。

第一次需要添加远程仓库名及本地仓库名，后续只需git push即可

参数

[remote-name] [tag版本号] 远程仓库名 origin，如果要打上tag， 推送时后直接跟版本号 即 git push origin v4.3（前提是已经打上标签）

–set-upstream origin [remote-name] 推新建分支到远程仓库（名称：remote-name）

13.git tag

新建标签

轻量级标签后直接加版本号即 git tag [tag-name]

参数：

-a [tag-name] 用来指定标签的名字

-m [tag-说明] 用来指定对应的标签说明

[提交对象的校验和（或前几位字符）] 用来后期加注标签，直接-a后跟校验和即可

分支模型

14.git branch

查看分支

参数：

[branch-name] 新建一个名为branch-name的分支

-d [branch-name] 删除一个名为brabch-name的分支

15.git checkout [branch-name]

切换到名为branch-name的分支

删除

16.git branch -d [local-branch]

删除一个已被终止的分支。

如果待删除的分支还没有转移到另外的分支上，Git会给出警告，并拒绝删除该删除操作。若坚持删除该分支，需要使用-D选项

git branch -D [local-branch]

17.git push origin –delete [remote-brance]

删除远程分支

windows下为 -d

ubuntu为 –delete

恢复

18.git branch [branch-name] [branch-hash-val]

恢复已知哈希值的分支。若不知，可通过git reflog找到哈希值。
